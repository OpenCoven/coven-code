# Coven Code Audit — June 2026

Comprehensive audit of the `coven-code` repo covering (1) repo health & branding, (2) UI/UX
enhancement opportunities, and (3) a slash-command reduction plan that focuses the surface on
commands aligned with the Coven.

**Scope:** v0.0.23, `main` @ `c095078`.
**Method:** full-codebase review of `src-rust/crates/{commands,tui,core,tools}`, `docs/`, `spec/`,
install/distribution scripts, and CI workflows.

## 3. UI/UX findings & roadmap

### What's working

- **Brand palette fully applied** — violet `#8B5CF6` primary / pink `#EC4899` accent throughout
  `theme_colors.rs`, including a deuteranopia theme and 8 selectable palettes.
- **Familiar system** — 7 animated familiars (kitty, nova, cody, charm, sage, astra, echo) with
  loading animations, F2 switcher, and Coven-daemon familiar loading. This is the strongest
  identity surface in the product.
- **Layout maturity** — 15+ overlays (rewind, tasks, help, history search, settings, theme picker,
  diff viewer, global search, familiar switcher, hooks menu…) with sane z-ordering.
- **Loading-state differentiation** — distinct "Running {tool}" / "Generating" / "Reasoning"
  states, stall detection (spinner turns red after 3 s).

### Prioritized enhancement roadmap

**P0 — fix now (shipped in this audit branch)**
1. "Coven Code Code Desktop" typo (B4).
2. `/feedback`→ OpenCoven issues URL (B2).

**P1 — high-impact polish (issues filed)**
3. **Complete the mascot rebrand**: `rustle.rs` → `mascot.rs`, `RustlePose` → `CompanionPose`,
   rename `public/Rustle.png` assets (COVEN.md pending item).
4. **WCAG contrast**: muted text `rgb(110,110,118)` on black ≈ 2.5:1 — below AA (4.5:1). Audit all
   muted/dim text colors.
5. **Spinner flash rate**: 12 frames at 50 ms ≈ strobe; slow to ≥100 ms/frame for WCAG 2.1
   three-flashes compliance.
6. **Keybinding discoverability**: F2 (familiar switcher), Alt+H (help), Ctrl+B (branch), and Tab
   (build/plan/explore cycling) are hinted nowhere — add to footer, help overlay, and onboarding

   ("Waiting on network…", "Permission pending…") and a stall message with a Ctrl+C hint.

   F1/F2 keys.
9. Typeahead suggestions: show source badges (command / history / context).

11. Remove legacy duplicate overlays (`render_simple_help_overlay`, `render_legacy_history_search`).
12. Familiar idle animations (blink, sway) and per-familiar eye-socket palette.
13. Cap welcome box width (~120 cols) on large terminals.

## 4. Slash-command reduction

### Problem

92 slash commands + 15 CLI commands (~5,900 lines). 20 are undocumented. Six redundancy clusters
make the surface hard to learn, and ~10 commands are dead Anthropic product hooks. The Coven
identity (daemon, familiars, rituals, multi-provider freedom) is diluted across a surface
inherited wholesale from Claude Code.

### Principles (approved 2026-06-10)

1. **Deep cut** toward a core surface (~40 commands), removing rather than hiding.
2. **Dead upsells removed entirely.**
3. **Novelty modes reframed** into Coven identity rather than deleted.
4. One command per concept; variants become arguments.

### Disposition matrix

**REMOVE — dead product hooks (9):**
`/passes`, `desktop`, `mobile`, `stickers`, `install-github-app`, `/web-setup`(`remote-setup`),
`/install-slack-app`, `/privacy-settings`, plus the hidden `/report` duplicate.

**REMOVE — redundant / low-value (18):**

| Command | Rationale / where it folds |
|---|---|
| `/stats` | duplicate of `/cost` |
| `/extra-usage` | folds into `/usage` |
| `/rate-limit-options` | folds into `/usage` |
| `/ctx-viz` | folds into `/context` |
| `/insights` | niche; `/usage` + `/export` cover it |
| `/heapdump` | developer diagnostic; `/doctor` covers health |
| `/release-notes` | `/version` + `/update` cover it |
| `/links` | low-value URL lister, undocumented |
| `/files` | low-value, conversation file list visible in context |
| `/summary` | overlaps `/compact` |
| `/teleport` | 390-line experiment overlapping `/share` + `/export` |
| `/remote-control`, `/remote-env` | Bridge surface for the removed remote/mobile products; revisit via `/coven` when the daemon grows remote control |
| `/checkpoints`, `/snapshot` | listing/diff folds into `/rewind` + `/revert` |
| `/accounts` | `/switch` with no args lists accounts |
| `/color-set` | internal helper of `/color` |
| `/rename` | becomes `/session rename <name>` |

**MERGE — variants become arguments (5 → into parents):**

| Removed | Surviving form |
|---|---|
| `/security-review` | `/review security` |
| `/ultrareview` | `/review ultra` |
| `/thinkback-play` | `/think-back play` |
| `/report` (hidden) | `/feedback` gains `report` alias |
| `/sandbox-toggle` | renamed: `/sandbox` primary, `sandbox-toggle` alias retained |

**REFRAME — coven-flavored (4 → 2):**

| Old | New | Notes |
|---|---|---|
| `/btw` | **`/whisper`** | "Whisper a side question to your familiar" — out-of-history Q&A |
| `/caveman`, `/rocky`, `/normal` | **`/incant <voice>` / `/incant off`** | Speech modes become incantations; voices: `caveman`, `rocky` (extensible) |

**KEEP — core surface (72 slash-invocable after Phase 1, down from 104; target ≈40 after Phase 2):**

- *Conversation & sessions:* `/clear` `/resume` `/session` `/fork` `/search` `/share` `/export`
  `/copy` `/compact` `/rewind` `/undo` `/revert`
- *Providers & auth:* `/connect` `/providers` `/login` `/logout` `/switch` `/refresh` `/model`
- *Modes:* `/plan` `/goal` `/thinking` `/think-back` `/effort` `/fast` `/output-style` `/incant`
  `/whisper` `/vim` `/voice`
- *Workflow:* `/commit` `/diff` `/review` `/memory` `/init` `/chrome` `/advisor`
- *Info:* `/help` `/status` `/context` `/cost` `/usage` `/version` `/doctor`
- *System:* `/config` `/permissions` `/sandbox` `/theme` `/color` `/keybindings` `/statusline`
  `/terminal-setup` `/update` `/feedback` `/import-config` `/exit`
- *Coven:* `/coven` `/familiar` `/agent` `/mcp` `/plugin` `/reload-plugins` `/skills` `/hooks`
  `/tasks` `/managed-agents`
- *CLI named commands kept:* `agents`, `agent`, `add-dir`, `branch`, `tag`, `ide`, `pr-comments`,
  `ultraplan`, `stats`

**Phase 2 (tracked as issues, not in this branch):** fold `/vim`, `/color`, `/statusline`,
`/terminal-setup` into `/config`; fold `/cost`+`/context` into `/usage`; unify
`/undo`/`/revert`/`/rewind` into a single rewind UI; evaluate `ide` integration reality. Target
end state: **~40 commands**.

### Flagged, kept pending verification

- `ide` (named) — IDE integration may be real via ACP; verify before removing.
- `/import-config` — imports `~/.claude` settings; strategically useful for migrating Claude Code
  users to Coven Code. Keep and document.
- `/managed-agents` — large (638 lines) and experimental, but OpenCoven-strategic. Keep.

### Upstream-merge tradeoff

Deleting commands in `crates/commands/src/lib.rs` will conflict with upstream Claurst merges in
that file. This is accepted: the command surface is a *user-facing* surface, which COVEN.md
designates as fair game for divergence. Keep deletions in focused commits to ease future
conflict resolution.
